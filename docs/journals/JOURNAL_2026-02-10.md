## RESULTAT CLE: COMBINED vs ML vs CYCLE

| Approche | Sharpe | Equity | MDD | Status |
|----------|--------|--------|-----|--------|
| **CYCLE seul** | 0.99 | 1.22x | ~15% | Baseline |
| **ML seul** | 0.12 | 1.23x | 4.4% | Mauvais Sharpe |
| **COMBINED** | **0.91** | **1.72x** | **5.8%** | PROMETTEUR |
| K3 (ref) | 0.99 | - | 13% | Reference |

### Analyse

**ML seul** (data/ML_directional.csv):
- Sharpe tres faible (0.12) - predictions pas assez precises
- Mais MDD excellent (4.4%) - evite les gros drawdowns
- Equity correcte (1.23x) sur 14 ans

**COMBINED** (CYCLE + ML):
- Sharpe 0.91 - proche de la reference K3/CYCLE
- Equity 1.72x - **+40% vs ML seul, +41% vs CYCLE seul**
- MDD 5.8% - toujours excellent
- 579 trades (vs 325 pour ML seul)

### Interpretation

La logique COMBINED fonctionne:
```python
if CYCLE == 0:  # bear market (halving phase)
    label = 0   # CASH force - protection
else:           # bull market
    label = ML_label  # utilise prediction ML
```

**Pourquoi ca marche:**
1. CYCLE protege des gros bear markets (d'ou MDD faible)
2. ML filtre les trades en bull market (d'ou plus de trades, meilleur Sharpe)
3. La combinaison capture le meilleur des deux mondes
